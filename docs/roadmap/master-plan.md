# Master-plan (каноническая дорожная карта)

> Версия: 2026-03-08 re-baseline after audit.
>
> Этот документ — каноническая дорожная карта реализации проекта строго по спецификации. Статусы ниже отражают не исторические optimistic-оценки, а фактическое состояние по совокупности кода, документации, UI и деплоя.

## Статусы этапов

Допустимые статусы:

- `not started`
- `in progress`
- `done`
- `blocked`
- `deferred`
- `changed with rationale`

## Правила выполнения плана

1. Нельзя объявлять этап `done`, если закрыт только каркас кода, но не закрыты UX, терминология, документация и тестовый контракт.
2. Любой новый промт на разработку обязан ссылаться на конкретный блок и подэтап master-plan.
3. Любое отклонение фиксируется в `docs/roadmap/progress-log.md`.
4. Любая новая функциональность должна сперва проходить проверку на соответствие `docs/product/spec-vnext.md` и `docs/product/canon/*`.
5. Legacy-решение без плана устранения не допускается.
