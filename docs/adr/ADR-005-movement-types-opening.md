# ADR-005: Типы движения и opening-семантика

## Контекст

Нужно устранить смешение учётных смыслов и закрепить канон движения как источника истины.

## Решение

- Тип движения описывает учётный смысл события, а не только знак.
- Базовые типы: `IN`, `OUT`, `ADJUST`, `OPENING`, `INVENTORY_APPLY`.
- `OPENING`, `ADJUST`, `INVENTORY_APPLY` не смешиваются.
- Начальное количество при создании позиции имеет два режима:
  - учётный приход (`IN`);
  - начало ведения учёта (`OPENING`).
- Создание позиции без количества не создаёт движение.
- Opening через импорт и opening через стартовую инвентаризацию имеют одинаковую природу стартового состояния.
- Инвентаризация поддерживает минимум два режима интерпретации результата:
  - с влиянием на аналитику;
  - как коррекция остатков без влияния на приходно-расходные отчёты.

## Последствия

- Учётная история становится однозначной и аудируемой.
- Отчётность может корректно различать стартовые и корректирующие события.
