# ADR-004: Канонические write-flow и read-model

## Контекст

Требуется единый шаблон выполнения записывающих операций и единая рамка построения представлений чтения.

## Решение

- Зафиксированы четыре класса write-flow:
  1. формирование/изменение сущностей;
  2. учётные события;
  3. синхронизационные сценарии;
  4. управляющие сценарии.
- Единый шаблон write-flow:
  1. приём команды;
  2. проверка допустимости;
  3. сборка канонического намерения;
  4. полная валидация;
  5. фиксация канонического изменения;
  6. перестроение зависимых представлений;
  7. аудит и понятный результат.
- Классы read-model/проекций: рабочие, состояния пространства, исторические, аналитические, управляющие, сигнальные.

## Последствия

- Уменьшается разброс сценариев обработки и повышается предсказуемость поведения системы.
- Проекции перестают быть самостоятельным источником истины.
