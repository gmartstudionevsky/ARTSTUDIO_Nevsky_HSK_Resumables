# ADR-001: Позиция учёта / Каталог позиций / Справочники

## Контекст

Нужно зафиксировать ядро предметной модели, чтобы исключить конкурирующие названия и локальные трактовки.

## Решение

- Главная сущность: **Позиция учёта**.
- Реестр сущностей: **Каталог позиций учёта**.
- Поддерживающие классификаторы: **Справочники**.
- Термин «номенклатура» сохраняется только как исторический/переходный.

## Последствия

- Терминология ядра становится единообразной в документации и следующих этапах реализации.
- Снижается риск конфликтов между UI, доменной моделью и отчётностью.
