# Канонические write-flow

## Единый шаблон

Каждый write-flow обязан проходить единый шаблон:

1. Приём команды.
2. Проверка допустимости.
3. Сборка канонического намерения.
4. Полная валидация.
5. Фиксация канонического изменения.
6. Перестроение зависимых представлений.
7. Аудит и понятный результат.

## Классы write-flow

- формирование и изменение сущностей;
- учётные события;
- инвентаризационные apply-сценарии;
- sync-сценарии;
- управляющие сценарии.

## Каноническая семантика событий

Базовые типы:

- `IN`
- `OUT`
- `ADJUST`
- `OPENING`
- `INVENTORY_APPLY`

### Первичное количество

При создании позиции с количеством должны поддерживаться два смысловых режима:

- **Учётный приход** → создаётся `IN`.
- **Начало ведения учёта** → создаётся `OPENING`.

Создание позиции без количества не создаёт движения.

### Opening по каналам ввода

Opening — это не способ ввода, а смысл сценария.

- opening новой позиции;
- opening через импорт;
- opening через стартовую инвентаризацию

имеют общую природу: признание стартового состояния учёта.

### Inventory apply

`INVENTORY_APPLY` относится к сценарию, где учёт уже вёлся, а инвентаризация приводит пространство к факту.
